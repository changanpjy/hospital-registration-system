# 运行指南

## 环境要求

- JDK 8 或以上
- Maven 3.6 或以上
- MySQL 8.x
- Node.js 18 或以上

本机测试可使用 Node.js 20。若命令行提示 `'npm' 不是内部或外部命令`，说明 Node.js 目录尚未加入 `Path`，可先重新打开终端，或使用完整路径执行 npm，例如：

```powershell
D:\node-v20.17.0-x64\npm.cmd install
D:\node-v20.17.0-x64\npm.cmd run dev
```

## 1. 初始化数据库

```bash
cd backend
mysql -uroot -p < 数据库初始化脚本/init.sql
```

脚本会创建 `hospital_registration` 数据库、核心业务表、演示账号、科室、医生、未来 7 天号源和 20 条仿真预约记录。

如果 MySQL 密码不是 `123456`，需要同步修改 `backend/src/main/resources/application.properties` 中的：

```properties
spring.datasource.username=root
spring.datasource.password=123456
```

## 2. 启动后端

确认 `backend/src/main/resources/application.properties` 中的数据库连接正确后执行：

```bash
cd backend
mvn spring-boot:run
```

后端地址：`http://localhost:8081`。

## 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端地址：`http://localhost:5173`。

前端使用 Vue 3 + TypeScript + Vite，开发服务会把 `/api` 请求代理到后端。后端和前端需同时启动，才能完整走通接口流程。

## 5. 主流程验证

1. 使用患者账号登录。
2. 在“就诊人”中维护本人或家属信息。
3. 进入“挂号预约”，选择科室。
4. 选择医生。
5. 选择未来 7 天内可预约号源。
6. 选择就诊人并提交预约。
7. 系统进入“预约成功”页面，展示预约详情、预约号和注意事项。
8. 在“我的预约”中查看预约记录。
9. 预约创建 30 分钟内可取消，超过 30 分钟后后端拒绝取消。
10. 在“个人中心”中点击“编辑信息”，可修改用户名、手机号和邮箱，保存后恢复只读展示。
11. 使用管理员账号登录，可进入“管理统计”查看统计数据，并在全部科室中选择科室、医生后维护排班号源。
12. 管理员可进入“用户管理”查看用户列表，并启用或禁用用户；当前管理员账号不能被禁用。
13. 普通用户登录后不会显示“管理统计”和“用户管理”模块。

## 6. 构建和检查

前端类型检查：

```bash
cd frontend
npm run typecheck
```

前端生产构建：

```bash
cd frontend
npm run build
```

后端测试：

```bash
cd backend
mvn test
```

## 7. 常见问题

- `npm` 找不到：检查 Node.js 是否安装并加入系统环境变量。
- 前端页面能打开但接口失败：确认后端服务已启动，并且端口是 `8081`。
- 数据库连接失败：确认 MySQL 已启动、账号密码正确，并已执行初始化脚本。
- 预约提交失败：检查号源是否已约满、停诊、超过当天预约截止时间，或同一就诊人同一科室同一天是否已经预约。
